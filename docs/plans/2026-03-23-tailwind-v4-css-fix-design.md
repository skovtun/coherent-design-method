# Tailwind v4 CSS Compatibility Fix

**Goal:** Fix the CSS runtime layer so all shadcn/ui components render correctly in Tailwind v4 projects. The platform already detects Tailwind version; the v4 code path has gaps that break visual rendering.

**Tech Stack:** TypeScript, Vitest, CSS

---

## Problem

`generateV4GlobalsCss()` in `tailwind-version.ts` produces an incomplete `@theme inline` block. `fixGlobalsCss()` in `fix-globals-css.ts` overwrites `app/globals.css` with this incomplete template at the end of every `coherent chat` run (`chat.ts:836-840`). Result: shadcn components that depend on missing CSS variables render broken.

### Missing from `@theme inline`

| Variable | Impact |
|----------|--------|
| `--color-transparent: transparent` | `border-transparent` on TabsTrigger, Select, and many components |
| `--color-black: #000` | Sheet overlay (`bg-black/50`) |
| `--color-white: #fff` | Destructive buttons/badges (`text-white`) |
| `--color-sidebar-*` (10 vars) | Entire Sidebar component |
| `--color-chart-1..5` | Chart components |
| `--radius-xs` | Sheet (`rounded-xs`) |

### Missing from `:root` / `.dark`

Sidebar variables (`--sidebar-background`, `--sidebar-foreground`, etc.) and chart variables (`--chart-1..5`) are generated by `buildCssVariables()` in core but NOT included in `generateV4GlobalsCss()`.

### `getThemeBlock()` exists but is unwired

`ShadcnProvider.getThemeBlock()` (`shadcn-provider.ts:606-620`) produces correct sidebar/chart `@theme inline` mappings. It is only called in tests, never in the chat/preview flow.

### Zero test coverage

No tests exist for `isTailwindV4`, `generateV4GlobalsCss`, `needsGlobalsFix`, or `fixGlobalsCss`.

## Design

### 1. Complete `generateV4GlobalsCss` (tailwind-version.ts)

Add all missing variables to the `@theme inline` block:

```
--color-transparent: transparent;
--color-black: #000;
--color-white: #fff;
--color-sidebar-background: var(--sidebar-background);
--color-sidebar-foreground: var(--sidebar-foreground);
--color-sidebar-primary: var(--sidebar-primary);
--color-sidebar-primary-foreground: var(--sidebar-primary-foreground);
--color-sidebar-accent: var(--sidebar-accent);
--color-sidebar-accent-foreground: var(--sidebar-accent-foreground);
--color-sidebar-border: var(--sidebar-border);
--color-sidebar-ring: var(--sidebar-ring);
--color-sidebar-muted: var(--sidebar-muted);
--color-sidebar-muted-foreground: var(--sidebar-muted-foreground);
--color-chart-1: var(--chart-1);
--color-chart-2: var(--chart-2);
--color-chart-3: var(--chart-3);
--color-chart-4: var(--chart-4);
--color-chart-5: var(--chart-5);
--radius-xs: 0.125rem;
```

Add corresponding `--sidebar-*` and `--chart-*` base variables to the `:root` and `.dark` blocks. Source values from the `config` parameter (the `DesignSystemConfig` already carries sidebar tokens via `buildCssVariables`).

### 2. Update `needsGlobalsFix` (fix-globals-css.ts)

Current check only looks for `--color-transparent` and `@theme inline`. Add checks for all critical variables so existing projects get fixed on next `coherent chat` run:

```typescript
const REQUIRED_V4_TOKENS = [
  '--color-transparent',
  '--color-sidebar-background',
  '--color-chart-1',
  '--color-black',
  '--color-white',
  '--radius-xs',
]
// If any required token is missing, return true
```

### 3. Add CSS validation function (new: css-validator.ts)

Create `validateGlobalsCss(projectRoot): string[]` that returns a list of missing/incorrect CSS variables. Called by `coherent preview` to warn the user.

Checks:
- Every `--color-*` in `@theme inline` has a corresponding `--var` in `:root`
- All `REQUIRED_V4_TOKENS` are present
- No stale v3 directives (`@tailwind base`) in v4 projects

This is a read-only check — it warns but does not modify files.

### 4. Test coverage (tailwind-version.test.ts, fix-globals-css.test.ts)

Tests for `generateV4GlobalsCss`:
- Contains all required `--color-*` tokens
- Contains sidebar variables in `:root` and `.dark`
- Contains chart variables
- Contains `--radius-xs`
- Contains `--color-transparent`

Tests for `needsGlobalsFix`:
- Returns `true` for v4 project missing `--color-transparent`
- Returns `true` for v4 project missing `--color-sidebar-background`
- Returns `false` for complete v4 globals
- Returns `true` for v3-style globals in v4 project

Tests for `isTailwindV4`:
- Detects `@tailwindcss/postcss` in deps
- Detects `tailwindcss: "^4"` in deps
- Detects `@import "tailwindcss"` in globals.css
- Returns false for v3 project

Tests for `validateGlobalsCss`:
- Catches missing tokens
- Returns empty array for complete file

### 5. Quality validator — catch junk classes (quality-validator.ts)

Add to `autoFixCode()`:
- Remove `className="-0"` (known AI generation artifact)
- Already handles `border-0` on TabsTrigger — verify it works
