# Story 3.4: Button & Interactive Element Quality

## Status

Draft

---

## Story

**As a** user,  
**I want** generated pages to never use native `<button>`, `<select>`, or raw checkbox/input without Label,  
**so that** buttons and forms always use shadcn components and look consistent.

---

## Acceptance Criteria

- [x] No native `<button>` elements in any generated page
- [x] Validator catches native form elements and reports error

---

## Details (from Epic 3)

- Add to UI-SYSTEM-PROMPT: COMPONENT IMPORTS (mandatory) — NEVER native `<button>` (always Button from @/components/ui/button); NEVER native checkbox (always Checkbox or Switch); NEVER native `<select>` (always Select); NEVER raw `<input>` without Label/form pattern
- Add validator check: scan for native `<button>`, `<select>`, `<input type="checkbox">` in generated code → error

---

## Tasks / Subtasks

- [x] Add COMPONENT IMPORTS (mandatory) rules to modifier prompt
- [x] Add validator rule(s) in quality validator: flag native button, select, checkbox
- [x] Run validator in chat pipeline (or validate command) and fail/report on violations
- [ ] Verify AC

---

## Dev Notes

- Modifier: `packages/cli/src/agents/modifier.ts`. Validator: `packages/cli/src/utils/quality-validator.ts` or similar.
- Validator should scan page code (and possibly component code) for regex or AST patterns.
