# Story 3.8: Figma API Integration

## Status

Draft

---

## Story

**As a** user with a Figma design file,  
**I want** to run `coherent import figma <file-url-or-key> --token <token>` and have the file structure fetched and stored,  
**so that** Coherent can later extract tokens, components, and pages from it.

---

## Acceptance Criteria

- [ ] `coherent import figma <url> --token <token>` fetches file data via Figma REST API
- [ ] Intermediate JSON saved (e.g. `coherent.figma-import.json`) with pages, components, styles, layout info
- [ ] Error handling: invalid URL, expired token, file too large
- [ ] Progress indicator during fetch

---

## Details (from Epic 3)

- Use Figma REST API: `GET /v1/files/:key` to fetch file structure
- Extract: pages and frames (→ app pages), components and component sets (→ shared components), styles (colors, typography, effects → tokens), auto-layout (→ flex/grid), text nodes (→ content)
- Store raw Figma data in `coherent.figma-import.json`
- Do NOT: pixel-perfect recreation, image export of frames, Figma plugin

**Alternative:** `coherent import figma --file ./export.fig` (local .fig if supported by API or tooling)

---

## Tasks / Subtasks

- [ ] Add `coherent import figma` command (subcommand under `import` or top-level)
- [ ] Integrate Figma REST API client (fetch file by key from URL or key)
- [ ] Parse response; write pages, components, styles, layout into intermediate JSON
- [ ] Handle errors (invalid URL, 401, file size) and show progress
- [ ] Document token requirement (Figma personal access token)

---

## Dev Notes

- Figma API: https://www.figma.com/developers/api#get-files-endpoint. File key from URL: figma.com/file/{key}/...
- Token: Figma → Settings → Personal access tokens.
