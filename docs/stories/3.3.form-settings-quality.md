# Story 3.3: Form & Settings Quality

## Status

Draft

---

## Story

**As a** user generating a settings or form page,  
**I want** the output to use two-column layout, proper Label+Input+description groups, Switch for toggles, and a Danger Zone,  
**so that** the page looks professional and scores ≥8/10.

---

## Acceptance Criteria

- [x] `coherent chat "add settings page"` → generates two-column form with proper labels, switches, and danger zone
- [x] All form inputs use shadcn components (not native HTML)
- [x] Settings page scores ≥8/10 on visual quality

---

## Details (from Epic 3)

**Fix:** Update settings template and UI-SYSTEM-PROMPT with form patterns:

- Two-column layout on desktop (`md:grid-cols-2`), form groups with Label + Input + description, Switch for toggles, Danger Zone at bottom
- FORM LAYOUT: Label above Input; space-y-2 within field group; space-y-6 between groups; CardFooter for actions; two columns on desktop for related fields; single column on mobile; Switch for booleans; Select for 3+ options, Radio for 2-3
- All form components: shadcn `<Switch>`, `<Select>`, `<Checkbox>`, `<RadioGroup>` — not native HTML
