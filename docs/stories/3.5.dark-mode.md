# Story 3.5: Dark Mode

## Status

Draft

---

## Story

**As a** user,  
**I want** to run `coherent chat "switch to dark mode"` or `"add dark mode toggle"` and get correct dark theme and an optional toggle,  
**so that** the app supports dark mode and all pages look correct (no white-on-white, no invisible text).

---

## Acceptance Criteria

- [x] `coherent chat "switch to dark mode"` → updates CSS variables (dark mode)
- [x] `coherent chat "add dark mode toggle"` → adds toggle (shared widget), toggles `dark` class on `<html>`
- [ ] All generated pages look correct in dark mode (semantic tokens: bg-background, text-foreground)

---

## Tasks / Subtasks

- [ ] Ensure dark mode variables exist in globals.css template (`.dark { ... }`)
- [ ] Verify all templates use semantic tokens (no hardcoded colors)
- [ ] Implement chat intent: "switch to dark mode" → update globals.css (or design-system.config + regeneration)
- [ ] Implement: "add dark mode toggle" → create shared widget (sun/moon), toggle `dark` on `<html>`, wire into layout
- [ ] Manual: preview in both modes, fix any white-on-white or invisible text

---

## Dev Notes

- Dark mode usually: `html.dark` + CSS variables for dark theme. Components use semantic tokens so they adapt.
- Toggle: small component that sets/removes `dark` on document.documentElement or `<html>`.
