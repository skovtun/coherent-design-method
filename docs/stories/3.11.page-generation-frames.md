# Story 3.11: Page Generation from Figma Frames

## Status

Draft

---

## Story

**As a** user who has imported a Figma file,  
**I want** each top-level frame to become a page in `app/` with layout derived from auto-layout and component instances,  
**so that** the prototype has the same structure as the Figma file.

---

## Acceptance Criteria

- [ ] Each Figma frame → a page in `app/`
- [ ] Layout derived from auto-layout properties (flex/grid, gap)
- [ ] Component instances use shared components (from 3.10)
- [ ] Pages build and render correctly

---

## Details (from Epic 3)

- Top-level frame = page. Frame name → route (e.g. "Dashboard" → `/dashboard`).
- Frame content: auto-layout direction → flex-col/flex-row; gap → gap-X; child frames → sections; component instances → shared imports; text nodes → content; image fills → placeholder or import if available.
- Patterns: nav at top → shared Header (CID-001); links at bottom → shared Footer (CID-002); sidebar → sidebar layout; centered card → auth layout.

---

## Tasks / Subtasks

- [ ] Read frames from intermediate JSON (after 3.8)
- [ ] Map each top-level frame to route and page path (app/{route}/page.tsx)
- [ ] Derive layout (flex/grid) from auto-layout; generate section structure
- [ ] Resolve component instances to shared components; generate imports and JSX
- [ ] Map text and images; handle placeholders
- [ ] Detect Header/Footer/sidebar patterns and use shared layout where applicable
- [ ] Ensure generated pages build and render
- [ ] Verify AC

---

## Dev Notes

- Depends on 3.8 (data), 3.10 (normalized components). Page generator should output Next.js App Router pages (page.tsx).
