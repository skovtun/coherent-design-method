# Story 3.10: Component Normalization (Figma → Coherent)

## Status

Draft

---

## Story

**As a** user who has imported a Figma file,  
**I want** Figma components to be normalized to Coherent base components (Button, Card, Input) or new shared components (CID-XXX),  
**so that** the app uses consistent, valid TSX with Tailwind.

---

## Acceptance Criteria

- [ ] Figma components with variants → shared components with typed props
- [ ] Common components (Button, Card, Input) → mapped to base library
- [ ] Unique components → new shared components with CID
- [ ] Component code is valid TSX using Tailwind classes

---

## Details (from Epic 3)

- Figma component = reusable element with variants. Map: Name → component name + CID; Variants → props; Auto-layout → flex/grid; Text → default props; Nested → imports.
- Normalize: Figma "Button" / "btn" → base `<Button>` with variants; "Card" → base Card + CardHeader + CardContent; "Input" / "TextField" → base Input; unknown → new shared CID-XXX.
- Smart detection by name and structure. Output: valid TSX, Tailwind, shadcn-style where applicable.

---

## Tasks / Subtasks

- [ ] Read Figma component data from intermediate JSON
- [ ] Implement mapping: Button/Card/Input-like → base component + variant props
- [ ] Implement: unique components → generate shared component file + register in manifest
- [ ] Generate TSX with Tailwind; ensure valid and buildable
- [ ] Verify AC

---

## Dev Notes

- Depends on 3.8 (data), 3.9 (tokens for styling). Uses Epic 2 shared component manifest and file layout.
- Base components: `components/ui/*` (shadcn). Shared: `components/shared/*`, coherent.components.json.
