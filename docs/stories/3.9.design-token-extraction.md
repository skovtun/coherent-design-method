# Story 3.9: Design Token Extraction (Figma → Coherent)

## Status

Draft

---

## Story

**As a** user who has imported a Figma file,  
**I want** Figma color and text styles to be mapped to Coherent design tokens (CSS variables, typography scale),  
**so that** the generated app reflects the Figma design system.

---

## Acceptance Criteria

- [ ] Named Figma color styles → CSS variables in globals.css
- [ ] Named Figma text styles → typography tokens
- [ ] Unnamed but frequent colors/fonts → proposed token assignments
- [ ] design-system.config.ts (or equivalent) reflects imported tokens

---

## Details (from Epic 3)

| Figma | Coherent Token |
|-------|----------------|
| Color styles (Primary, Secondary...) | `--primary`, `--secondary` in globals.css |
| Text styles (Heading 1, Body...) | Typography scale in tailwind config |
| Effect styles (Shadow S, M, L) | Box shadow tokens |
| Corner radius (from components) | `--radius` variable |
| Auto-layout gaps | Spacing scale |

Smart mapping: Primary/Default → `--primary`, Primary/Hover → primary + hover opacity; Neutral/50–900 → muted scale; Danger/Error → `--destructive`. If no named styles → extract most-used and propose names.

---

## Tasks / Subtasks

- [ ] Read `coherent.figma-import.json` (or equivalent) for styles
- [ ] Map color styles to CSS variables; write/update globals.css
- [ ] Map text styles to typography scale (tailwind or config)
- [ ] Map effects to shadow tokens, radius to --radius, gaps to spacing
- [ ] Propose tokens for unnamed frequent values
- [ ] Update design-system.config or token source of truth
- [ ] Verify AC
