# v0.6.94 — Quality Overhaul

## Context

E2E testing of Projector tutorial (v0.6.88→0.6.93) revealed 5 bugs and significant gaps in quality rules compared to ui-ux-pro-max-skill (67K stars, 161 reasoning rules). Atmosphere extraction (v0.6.92) proved the constraint system works when directives reach the AI — now we apply the same rigor to accessibility, interaction states, and performance.

## Goals

1. Fix all 5 known bugs from E2E testing
2. Add 9 new quality rules (closing gaps vs competitor)
3. Wire atmosphere.primaryHint → actual design token regeneration
4. Add lightweight mood/industry presets for common app types

## Non-goals

- Figma MCP integration (external dependency, separate project)
- Full 161-rule industry reasoning engine (post-PH, data-heavy)
- Visual canvas / GUI (we stay CLI-first)

## Part 1: Bug Fixes (5)

### 1.1 primaryHint not in atmosphere directive
- **File:** `packages/cli/src/commands/chat/plan-generator.ts` — `renderAtmosphereDirective()`
- **Bug:** `atmosphere.primaryHint` is extracted and stored but never emitted in the directive block
- **Fix:** Add line: `- Primary: Use ${primaryHint} tones for --primary token. Buttons, links, active states should reflect this.`

### 1.2 Negative dates "-2341m ago"
- **File:** `packages/cli/src/agents/design-constraints.ts` — formatRelative helper (~line 247)
- **Bug:** `Date.now() - futureDate` produces negative minutes, renders "-2341m ago"
- **Fix:** Add `if (d < 0) return new Date(iso).toLocaleDateString('en-US', { month: 'short', day: 'numeric' })` as first guard

### 1.3 --page refine fails ~50%
- **File:** `packages/cli/src/commands/chat/utils.ts` — `resolveTargetFlags()` (~line 273-295)
- **Bug:** When target page file doesn't exist on disk, prompt sends empty code block. AI interprets inconsistently.
- **Fix:** If `!currentCode`, replace with `"(page not yet generated — create from scratch based on the request)"` instead of empty tsx block

### 1.4 FilterBar contract drift
- **File:** `packages/cli/src/commands/chat/split-generator.ts` — `filterManifestForPage()` (~line 59-71)
- **Bug:** Filtered manifest includes stale propsInterface from manifest, not plan's authoritative version
- **Fix:** After filtering by name/usedBy, overlay plan's propsInterface onto matched manifest entries before returning. Plan is source of truth for props contract.

### 1.5 Plan sharedComponents cap 8 rejects on refine
- **File:** `packages/cli/src/commands/chat/plan-generator.ts` — `ArchitecturePlanSchema` (~line 137)
- **Bug:** `.max(8)` rejects plan updates when existing plan already has 8 components + AI adds more
- **Fix:** Raise to `.max(12)`. Update PLAN_SYSTEM_PROMPT to say "Maximum 12 shared components". Update test that checks cap.

## Part 2: New Quality Rules (9)

### Two-layer placement

| Rule ID | Pre-gen (constraint) | Post-gen (validator) | Severity |
|---|---|---|---|
| MISSING_ARIA_LABEL | Both | Both | warning |
| LOW_CONTRAST | — | Post only | info |
| SMALL_TOUCH_TARGET | Both | Both | warning |
| NO_LOADING_STATE | Pre only | — | info |
| NO_ESCAPE_ROUTE | Pre only | — | info |
| EMOJI_IN_UI | Both | Both | warning |
| NO_FONT_DISPLAY | — | Post only | info |
| NO_REDUCED_MOTION | Both | Both | info |
| MISSING_ALT | Already in pre-gen | Post only (upgrade RAW_IMG) | warning |

### Pre-gen additions (design-constraints.ts)

Add to `CORE_CONSTRAINTS`:
```
ACCESSIBILITY (WCAG AA mandatory):
- Icon-only buttons/links MUST have aria-label describing the action
- Interactive elements: min 44×44px touch target (use p-2.5 or min-h-[44px] min-w-[44px])
- No emoji characters in UI text — use Lucide icons (vector, scalable, theme-aware)
```

Add to `INTERACTION_PATTERNS`:
```
LOADING STATES (async operations):
- Buttons that trigger async: disabled={loading} + spinner icon while pending
- Pattern: const [loading, setLoading] = useState(false); onClick: setLoading(true) → await action → setLoading(false)
- Never leave a button clickable during an async operation

ESCAPE ROUTES:
- Every Dialog/Sheet/Drawer MUST have: onOpenChange handler + visible close button (X)
- Esc key must close (shadcn default, but verify onOpenChange is wired)
```

Add to `DESIGN_QUALITY_COMMON`:
```
MOTION ACCESSIBILITY:
- All CSS transitions/animations MUST respect prefers-reduced-motion:
  @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }
- Add this block to globals.css during scaffold
```

### Post-gen additions (quality-validator.ts)

```typescript
// MISSING_ARIA_LABEL: icon-only interactive without aria-label
// Match: <Button ...><Icon /></Button> without aria-label
{ pattern: /<(?:Button|button|Link|a)\b[^>]*>[\s\n]*<(?:\w+Icon|[A-Z]\w+)\s[^>]*\/?>[\s\n]*<\/(?:Button|button|Link|a)>/,
  negative: /aria-label/,
  type: 'MISSING_ARIA_LABEL',
  message: 'Icon-only button/link without aria-label — add aria-label="description"',
  severity: 'warning' }

// SMALL_TOUCH_TARGET: interactive without sufficient sizing
// Check: Button variant="ghost" size="icon" without min-h-[44px] or p-2.5+
{ pattern: /size="icon"/,
  negative: /min-h-\[4[4-9]|min-w-\[4[4-9]|p-[3-9]|p-2\.5/,
  type: 'SMALL_TOUCH_TARGET',
  message: 'Icon button may be < 44px touch target — add min-h-[44px] or increase padding',
  severity: 'warning' }

// EMOJI_IN_UI: emoji unicode in JSX
{ pattern: /[\u{1F300}-\u{1F9FF}\u{2600}-\u{26FF}\u{2700}-\u{27BF}]/u,
  type: 'EMOJI_IN_UI',
  message: 'Emoji in UI — use Lucide icon instead (vector, scalable, theme-aware)',
  severity: 'warning' }

// NO_REDUCED_MOTION: animation/transition without prefers-reduced-motion
// Only check globals.css, not per-component (Tailwind handles inline)
// Checked separately in globals.css scan

// NO_FONT_DISPLAY: @font-face without font-display
// Checked in globals.css scan

// LOW_CONTRAST: basic check for text-muted-foreground on bg-muted (known low-contrast pair)
// Info-level only — full contrast checking needs runtime
```

## Part 3: Token Regeneration from primaryHint

### Problem
`atmosphere.primaryHint = "zinc"` is extracted but never modifies actual CSS tokens. Buttons stay blue.

### Solution
After plan generation (Phase 2), when `atmosphere.primaryHint` is set:
1. Map hint to hex color: `{ zinc: '#18181b', emerald: '#059669', indigo: '#4f46e5', rose: '#e11d48', amber: '#d97706' }`
2. Update `modCtx.config.tokens.colors.light.primary` and `.dark.primary` with the mapped color
3. This flows into `globals.css` regeneration at end of pipeline (already happens via `regenerateFiles`)

### File changes
- `packages/cli/src/commands/chat/split-generator.ts` — after atmosphere merge block, add token override
- New: `packages/cli/src/utils/color-presets.ts` — primaryHint → hex mapping + derived foreground colors
